PHP-Forms Documentation v0.1.4.1

This PHP class has been developed to make form generation easier for you. It also looks a hell of a lot tidier and allows you to combine simple forms and complex designs.

How do I use it?

Download the source for this repository, put it into a directory within your site. And because its a static class it's very easy to use. First include the file like so:

Example:

Here is an example of how to use the form class. This example is that of a comments form.

<?php
    //  We need the file, so include it.
    include 'form.php';

    //  Store the form options in an array. 
    $opts = array('name' => 'form1', 'action' => '', 'method' => 'POST', 'id' => 'comment');

    Form::start($opts);
        Form::input(array('name' => 'name', 'placeholder' => 'Enter your name here'));
        Form::textarea(array('name' => 'comment', 'value' => 'Enter your message here'));
        Form::submit();
    Form::end();
?>

The above example should output HTML like so:

<!-- START OF FORM1 FORM --><form name="form1" action="" method="POST" id="comment" ><input name="name" placeholder="Enter your name here" ><textarea name="comment" value="Enter your message here" >Enter your message here</textarea><input value="submit" type="submit" ></form><!-- END OF FORM -->

Methods:

Now here is a little bit of detail about each feature in this class, how to implement the methods, the parameters they take and what they output.

This class has been developed to try and give you complete freedom over how you want to generate your html forms. But It also provides you with some simple pre-generated forms, such as an upload form or a login form.

start()

Implementation

<?php
    Form::start();
?>

It's that easy. Because this class is so clever, if you don't enter any parameters it will provide you with default settings. This allows you to create html forms on the fly.

Parameters

Options

This parameter is the only parameter for this method. It is where you define your options for the form. E.g. its name, its action and its method. This parameter takes an array only. It requires a named key and a value.

Imagine that you are writing you html, and you want to set the name of an element, you'd type:

<form name="formname">

Well this parameter works in a similar way:

<?php
    Form::start(array('name' => 'formname'));
?>

And there you go! It works the same way for things like method, action etc... Most methods throughout this class take a similar parameter; an array where you define your html element properties.

End()

Implementation

This method is used to close the form. It basically adds </form> to the page.

<?php
    Form::start();
    //  Other stuff goes here
    Form::end();
?>

Parameters

This method does not take any parameters. I included it purely to make it tidier and more consistent when wanting to end the html form.

Dropdown()

Implementation

This method is available to make generating drop-down lists easier for you. This was one of the main reasons I made this class.

<?
    Form::dropdown(array('car' =>  'ferrari', 'plane' => 'boeing 747'), array('name' => 'vehiclelist', 'class' => 'dropdown'));
?>

Parameters

There are two parameters for this method, both being arrays. The first parameter is the list of items that you would like to be displayed and the values that they carry.

Items

<?php
    array('what_is_displayed' => 'value_of_item');
?>

Options

The second parameter is the options parameter, this acts the same as the options parameter for the start() method. Please see start() method.

Textarea()

This method was introduced to make implementing textareas with the form class easier.

Implementation

<?php
    Form::textarea(array('name' => 'my_textarea', 'class' => 'textarea1'), 'This will be inside the textarea when it is generated.');
?>

Parameters

Just like most of the methods in this class it takes an array of options. The other parameter is just a string, this is what will be shown inside the textarea when it is generated.

Input()

This is just a standard input field where you can determine the type. By default (no parameters) it is a text field.

Implementation

<?php
    Form::input(array('name' => 'first_name', 'placeholder' => 'Type your first name here.'));
?>

Parameters

This takes the same parameter as start(); an array of options.

Submit()

This method generates a submit button to finish off your form.

Implementation

<?php
    Form::submit(array('name' => 'my_submit_button'));
?>

Parameters

This can take an array of options (again). If you do not pass through any parameters the button will use default values.

Password()

Implementation

<?php
    Form::password();
?>

Parameters

This can take an array of options, but it is not necessary because the default values are probably sufficient.

Login()

This method is one of the special ones that I decided to include, it makes creating login forms easier, well one line. As of now this method is not easily customisable and uses the old style of parameters.

Implementation

<?php
    Form::login($name = 'login', $action = 'page2.php', $id = 'login-form', $class = 'login-form', $line-breaks = true);
?>

Parameters

Name

This parameter is the name of your login form. The default value is login.

Action

This parameter is the action of the form. This is where you are sending the POST data to.

Id

This is the id of the form. Used for CSS.

Class

This is the class of the form. Used for CSS.

Breaks

If this parameter is false there will be no line breaks between the inputs. If true then there will. The default value of this parameter is false.

GetForms()

This method returns all of the forms that you have on the page where you use this method.

Implementation

<?php
    Form::getForms();
?>

Parameters

This method does not take any parameters.

Upload()

This method generates an upload form and handles the movement of the file for you. If the action is empty it will move the temporary file to the location that you specify, else you will be required to place the recieveFile() method on the receiving page.

Implementation

<?php
   Form::upload('image_upload', '/uploads/images/', array('image/jpeg', 'image/png', 'image/gif', 'image/jpg'), '');
?>

Parameters

Name

This is the name of your upload form.

Where

This is where the file will be uploaded.

Allows

This is an array of file types that are allowed.

Action

This is the action of the form, if this is empty the upload logic will run on the same page as the form. If the action is not empty you will need to use the receiveFile() method on the same page as you specified in the action, or the page where you wish to handle the upload. This is handy If you wish to implement some form of captcha system when uploading files.

ReceiveFile()

Implementation

<?php
   Form::receiveFile();
?>

Parameters

This method does not take any parameters.
